## 목적
사용자가 "이 대본 잘 나왔다"고 판단할 때, 해당 대본의 성공 패턴을 추출해서 나중에 재사용할 수 있도록 저장한다.

## 실행 절차

### 1단계: 현재 대본 패턴 분석
현재 대화에서 가장 최근에 작성한 대본을 분석한다:
- 카테고리: [자동 감지 또는 사용자 입력]
- 훅 패턴: 어떤 방식으로 시작했는가
- 본문 구조: 어떤 3단 구조를 사용했는가
- CTA: 어떤 행동 유도 문구를 썼는가
- 길이/형식: 쇼츠·롱폼·상품 여부

### 2단계: 파일 저장
`.claude/skills/user-saved/SKILL.md`에 추가 (파일이 없으면 생성, 있으면 append)

저장 형식:
```markdown
## [날짜] [카테고리] 성공 패턴

### 훅
[사용된 훅 패턴과 공식]

### 본문 구조
[사용된 3단 구조 요약]

### CTA
[사용된 CTA 문구]

### 원본 대본 요약
[핵심 문장 1-2개만 발췌]

---
```

### 3단계: 저장 완료 알림
"[카테고리] 패턴이 .claude/skills/user-saved/SKILL.md에 저장됐습니다."

## 저장된 패턴 활용
- 다음 대화에서 같은 카테고리 대본 요청 시 저장된 패턴 참고
- "저장된 패턴 보여줘"라고 하면 user-saved/SKILL.md 내용 출력
- "저장된 패턴으로 만들어줘"라고 하면 해당 패턴 적용

## 주의사항
- 저장 전 사용자에게 카테고리명 확인
- 개인정보·민감한 정보가 포함된 대본은 저장 전 사용자 동의 확인
- 파일 크기 관리: 패턴이 20개 이상 쌓이면 정리 권고
